# 家庭资产规划大师

一个面向 `GitHub Pages` 免费部署的家庭资产规划 Web 项目。  
当前版本已经不是占位原型，而是可实际使用的前端应用：支持家庭资产、负债、收支、目标、投资组合、数据备份与诊断分析，并适配静态部署场景。

项目管理视角的当前进度可见：

- [PROJECT_STATUS.md](/Users/zhangyi/Documents/财务管理大师/PROJECT_STATUS.md)

## 当前能力

当前已完成：

- `Vite + React + TypeScript` 前端架构
- `HashRouter` 路由，适配 GitHub Pages 子路径和刷新场景
- 总览页：家庭资产总览、资产结构、趋势、目标进度、风险摘要、预算预警、投资联动
- 资产台账：录入、编辑、删除、搜索、筛选、排序、结构分布
- 负债管理：录入、编辑、删除、搜索、筛选、排序、杠杆摘要
- 收支管理：收入/支出录入、预算上限、分类图表、预算预警
- 目标计划：目标录入、目标清单、月投入测算、A/B 情景模拟
- 投资组合：持仓录入、CSV 导入导出、真实收益计算、再平衡建议、漂移图表
- 数据配置：家庭基础参数、JSON 导入导出、操作历史、快照历史
- 家庭诊断：综合评分、风险信号、优先动作、可跳转处理、诊断报告导出
- 本地持久化：浏览器刷新后仍保留数据
- GitHub Pages 自动部署工作流
- 移动端与窄屏适配基础能力

## 页面结构

当前导航包括：

- `总览`
- `资产台账`
- `分析管理`（当前对应负债管理）
- `收支管理`

## 本地开发

安装依赖：

```bash
npm install
```

本机开发：

```bash
npm run dev
```

如果要让同一局域网内的其他设备访问本机预览：

```bash
npm run dev:host
```

## 本地预览构建产物

构建：

```bash
npm run build
```

本地预览构建结果：

```bash
npm run preview
```

如果要在手机、平板或其他设备上访问构建后的页面：

```bash
npm run preview:host
```

## 数据说明

当前数据存储方式：

- 家庭数据：浏览器本地存储
- 支出预算上限：浏览器本地存储
- 图表与诊断：基于本地数据实时计算

支持的数据操作：

- 从 `数据配置` 页导出完整 `JSON` 备份
- 从 `数据配置` 页导入 `JSON` 备份
- 从 `投资组合` 页导入和导出持仓 `CSV`
- 从 `家庭诊断` 页导出诊断 `Markdown` 报告

## 家庭诊断

点击右上角 `开始诊断` 会进入诊断页，当前会输出：

- 综合评分与等级
- 五项维度评分
- 风险信号清单
- 优先动作建议
- 对应模块跳转入口

诊断结果会结合以下数据：

- 资产与负债
- 收支与预算
- 目标准备度
- 投资组合执行情况

## GitHub Pages 部署

项目已经包含自动部署工作流：

- [deploy.yml](/Users/zhangyi/Documents/财务管理大师/.github/workflows/deploy.yml)

当前部署策略：

- 使用 `Vite`
- `base: './'`
- 使用 `HashRouter`
- 推送到 `main` 分支时自动构建并发布到 `GitHub Pages`

启用方式：

1. 把仓库推送到 GitHub
2. 在仓库设置里启用 `GitHub Pages`
3. 选择使用 GitHub Actions
4. 后续推送到 `main` 会自动发布

## 适配与上传建议

为了以后继续上传、迁移或二次部署，当前已经做了这些兼容处理：

- 所有路由基于 `HashRouter`
- 站点资源路径尽量使用相对路径
- 增加了 `site.webmanifest`
- 补充了页面标题、描述、主题色和移动端 meta
- 提供了 `dev:host` / `preview:host` 用于多设备联调

如果后续要继续扩展，优先建议：

1. 增加真实用户引导与首次使用空状态
2. 增加更细的投资标的成本法与分红明细
3. 增加诊断结果的跨页高亮与自动聚焦
4. 增加多家庭档案切换
5. 如需多人共享，再考虑引入后端或云同步
