# 项目状态清单

最后更新：2026-04-23

## 当前版本定位

当前仓库已经从“静态原型”推进到“可运行、可录入、可分析、可部署”的前端产品版本，适合：

- 本地个人使用
- GitHub Pages 静态部署
- 继续做前端迭代
- 作为后续接后端/云同步的基础版本

## 已完成

### 基础工程

- `Vite + React + TypeScript`
- `HashRouter` 路由，适配 GitHub Pages
- GitHub Pages 自动部署工作流
- 构建产物输出到 `dist/`
- 支持多设备本地联调：`dev:host` / `preview:host`

### 数据层

- 统一家庭数据模型
- 浏览器本地持久化
- JSON 导入导出
- 财务快照历史
- 操作历史记录
- 数据校验与版本兼容

### 业务模块

- 总览页
- 资产台账
- 负债管理
- 收支管理
- 目标计划
- 投资组合
- 数据配置
- 家庭诊断

### 投资组合能力

- 持仓录入与编辑
- CSV 导入导出
- 真实净收益计算
- 分红收益率 / 交易成本率
- 按代码搜索与排序
- 组合再平衡建议
- 仓位漂移图表
- 与首页和资产台账联动

### 诊断能力

- 综合评分与等级
- 五项维度评分
- 风险信号识别
- 优先动作建议
- 可跳转到对应模块处理
- Markdown 报告导出

### 适配与部署

- GitHub Pages 路径兼容
- 相对静态资源路径
- `site.webmanifest`
- 页面标题、描述、主题色
- 手机和窄屏的基础适配

## 当前仍可继续增强

### 高优先级

- 诊断页跳转后的目标区域高亮
- 首次使用空状态与引导文案
- 更明确的模块命名
  当前导航中的 `分析管理` 实际对应负债管理，建议后续改名
- 更细的投资组合统计
  例如分红明细、手续费明细、交易批次

### 中优先级

- 多家庭档案切换
- 更丰富的筛选条件持久化
- 更完整的导出格式
  例如 PDF / Excel / 更结构化的报告
- 更多跨页联动
  例如从诊断页直接打开带预填表单的模块

### 长期方向

- 云同步 / 多端共享
- 用户登录
- 后端数据库
- 外部行情与资产价格同步
- 自动提醒 / 定时任务

## 当前已知限制

- 完全基于前端本地存储，不适合多人协作共享
- 投资标的最新价需要手动维护，不接外部行情
- 诊断为规则型前端分析，不是专业投资、税务或法律意见
- 没有账号体系，清理浏览器数据会影响本地记录

## 下一阶段建议

推荐按下面顺序继续：

1. 完成诊断结果的跨页高亮和自动定位
2. 增加首次使用引导与空状态
3. 修正导航命名与产品语言统一
4. 细化投资组合交易与分红记录
5. 评估是否接入后端或云同步

## 交付检查

当前交付前建议确认：

- `npm run build` 通过
- GitHub Pages 已启用
- `README.md` 与当前版本一致
- `PROJECT_STATUS.md` 已更新
- 重要数据已从 `数据配置` 页面导出备份
